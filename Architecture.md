## コアコンセプト

### スクラムを人間とAIの共通プロトコルとして使う

スクラムは「人間チームのためのフレームワーク」として生まれましたが、
このツールでは**人間とAIが協働するための共通言語**として再解釈します。

- 人間は「何を作るか」に集中する（プロダクトオーナー的役割）
- AIは「どう作るか・どう進めるか」を担う（開発チーム的役割）
- スクラムのプロセスが、その協働の透明性を担保する

## Tauri × AIエージェントのアーキテクチャ

AIエージェントがタスクを自律実行する際の処理境界を明確にします。

```
フロントエンド（React）
│  カンバン表示・進捗確認・人間の操作
│
│  invoke("execute_task", { task_id, instructions })
│
Rustバックエンド（Tauri）         ← 実際の処理はここで行う
├── AI API呼び出し（reqwest）
├── ファイル操作（std::fs）        ← バリデーション後に実行
├── コマンド実行（std::process）   ← 許可リスト制限あり
└── SQLite読み書き（sqlx）
```

**設計方針：フロントエンドに直接のシェル実行権限は与えない。**
AIの指示をRust側で受け取り、バリデーションを挟んでから実行することで、
AIの誤操作によるファイル破壊などのリスクを防ぎます。

## 進行中アクションの安全機構 (Interaction Guard)

スプリントタイマーやAIエージェントなどの「進行中のアクション」が存在する状態で、ユーザーがワークスペース（プロジェクト）を切り替えようとした場合、データの不整合や意図しない裏側での処理を防ぐための「Interaction Guard（安全機構）」を設けます。

### 設計方針

フロントエンドのUIコンポーネント（例: `ProjectSelector`）は、プロジェクトを切り替える前に、現在アクティブな処理がないかを各Contextからチェックします。

1. **状態の検知**: `SprintTimerContext` 等を用いて、タイマーが `RUNNING` 状態であるかを判定。将来的にはAIエージェントの処理中 (`isExecuting`) なども検知対象に含めます。
2. **ユーザーへの確認**: アクティブな処理が検出された場合、直ちに切り替えるのではなく、確認ダイアログ（`window.confirm`等）を表示します。
3. **安全な停止と切り替え**: ユーザーが切り替えに同意した場合、進行中の処理を安全に停止（あるいは一時停止）してから、`currentProjectId` の変更を行います。同意しない場合は切り替えをキャンセルします。

この仕組みにより、バックグラウンドで不要な処理が別のプロジェクトコンテキストで走り続けることを防ぎます。
