# Epic 22: Agent Orchestratorの統合と自律的タスク生成 (Walkthrough)

## 変更内容 (Changes)

本Epicでは、LLMのTool Calling機能（関数呼び出し機能）を用いて、チャットインターフェースでの曖昧な要求からAIが自律的にタスクを分解し、データベースへ登録・UIへ反映する「エージェント・オーケストレーター」基盤を実装しました。

- **バックエンドの実装 (Rust / Rig Tool)**
  - `rig-core` の `Tool` トレイトを実装した `CreateStoryAndTasksTool` を開発し、エージェント（Anthropic / Gemini）へ登録しました。
  - DB上に「ストーリー」とその「サブタスク」を同期的にトランザクションとして一括追加するデータ操作関数 `insert_story_with_tasks` を実装しました。
  - ツールが実行され、データベースの更新が完了した直後に、Tauri Eventである `app.emit("kanban-updated", ())` を発火させる仕組みを構築しました。

- **フロントエンドの改修 (React)**
  - バックエンドとフロントエンドを疎結合に保つため、`ScrumContext.tsx` 内において `@tauri-apps/api/event` を用いたイベントリスナー (`listen("kanban-updated")`) を実装しました。
  - バックエンドから発行された更新イベントを受信すると自動的に既存のフェッチ機構（`fetchStories`, `fetchTasks` など）がコールされ、リロードなしでカンバンボードが最新状態になるよう改修しました。

## テスト内容・結果 (Validation Result)

- **ビルド・コンパイルチェック**: Rust側コード（`ai.rs`, `ai_tools.rs`, `db.rs`, `rig_provider.rs`）について `cargo check` によるビルドをパスし、RigによるTool Callingの型解決を含めて安全性が保たれていることを確認済みです。
- **IPCとエラーハンドリング**: Rigのエラー型に互換性を持たせるため `td::error::Error` を実装し、Toolの実行で発生する可能性のあるエラーがフロントへ確実に伝達または安全に処理されることを保証しています。

## PO（人間）による確認手順 (Manual Testing Steps)

いよいよ魔法のようなタスク自動生成を体験する時です。以下の手順に沿って動作確認を行ってください。

1. 起動中（`npm run tauri dev`）の「MicroScrum AI」デスクトップアプリのウィンドウを開きます。
2. 画面右端の「AIサイドバー」を開きます（閉じていた場合）。
3. チャット欄に、以下のような機能要望（システムの仕様追加など、やや曖昧な要件）を入力して送信してください。
   > **入力例:**  
   > 「プロフィールのパブリック公開画面を追加し、それに伴うアイコン画像の登録と自己紹介文の更新ができるようにしたい。実装に必要なタスクを一括で起票して！」
4. 少しの間、AIからの回答（Tool機能の呼び出し・実行結果処理）を待ちます。
5. AIが「タスクを登録しました」と完了報告のメッセージを返してくるのを確認します。
6. 返答が来た直後に、画面左側・中央の「カンバンボード（プロダクトバックログ）」に視線を移します。
7. **手動でのページリロード（Ctrl+R など）を行うことなく、新規のストーリーと複数のサブタスクがカンバン上にフワッと（即時に）出現すること** を確認します。

以上の手順により、要求から要件定義、タスク分解までが自動化された「次世代エージェント基盤」の動作検証が完了となります！
