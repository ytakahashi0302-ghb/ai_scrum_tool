# Epic 24: ClaudeCLIを用いたタスクの自律実行基盤の構築

本実装は、カンバン上のタスクを起点に、ClaudeCLI（Claude Code）を外部プロセスとして起動し、コーディング等の開発作業を自律的に実行させる基盤を構築する。

## 要件の技術アプローチ

### 1. プロセス管理戦略 (PTY vs 標準出力パイプ)
ClaudeCLI は、ファイル書き込み前のY/N確認、プロンプト選択、プログレスバーの表示など、ターミナル上のインタラクティブな出力やエスケープシーケンスを多用するツールです。そのため、**PTY（仮想ターミナル）経由での起動と管理を強く推奨**します。

*   **PTY採用の理由**:
    *   ANSIエスケープシーケンス（文字色やカーソル移動制御）がそのままフロントに引き渡されるため、ターミナルコンポーネント（xterm.jsなど）で描画した際にデザインが崩れにくく、リアルタイムでリッチな出力が可能です。
    *   単なる標準出力ストリーミング (`stdout` パイプ) だと、対象プラットフォームが非TTY環境であると認識し、ClaudeCLIの動作仕様が変わる（意図した対話フォーマットが出力されないなど）リスクがあります。
    *   将来的にどうしてもユーザー側から割り込み（Ctrl+CやYes/Noの入力）を送る必要が生じた際、PTY通信であれば対応が容易になります。すでにEpic 21でPTY通信バグを解消している実績があるため、その基盤を再利用できます。

### 2. バックエンド（Rust）実装の設計
*   **プロセス起動・パラメータ構成**: タスク情報の詳細（タイトル、説明など）を抽出し、ClaudeCLIのコマンドライン引数、もしくは初期プロンプトとして渡します。
*   **ログの同期**: PTYから一定バッファサイズごとに読み取ったチャンクデータを Tauri イベント（例: `app_handle.emit_all("claude_cli_output", payload)`）でストリーミング配信します。
*   **ガードレール**: 実行ディレクトリを必ず「対象のプロジェクトルート」に限定。さらに長期間のハングを防止するため、デフォルトで **3分（180秒）のタイムアウト** を設定しプロセスを強制終了する安全機構をRust側に設けます。APIトークン費用が膨大になるリスクを防ぎます。
*   **手動強制終了（Kill）機能**: ユーザーがUI上で暴走を検知した際に即座にプロセスを停止できるよう、ターミナルUIに「強制停止」ボタンを配置し、後述するRustのKill処理と連動させます。

### 3. フロントエンド（React）UI統合
*   **トリガーUI**: カンバン上のタスクカード、もしくはタスク詳細モーダル内に「開発を実行（Launch Claude）」のトリガーボタンを配置します。
*   **ターミナルUI連携**: 既に構築済みの Terminal Dock 領域を活用し、Tauri イベントをリッスンして逐次ログを表示します。これに加え、「強制停止」ボタンを上部などに配置します。

### 4. 実行後の処理と状態管理
*   **自動ステータス遷移**: トリガーを押した時点でタスクステータスを `In Progress`（または実行中独自の内部フラグ）に変更。正常終了すれば、手動でのPull Requestやコードレビューに進めるため `Review` へ自動遷移させます。

## Verification Plan

### 制約下でのテスト
*   **MOCKスクリプトによるテスト**: 実際に毎回APIトークンを消費するのを防ぐため、擬似的・定期的に出力を吐出し、指定秒数で自発的に終わるダミーのsh/batスクリプトをRustから起動し、フロントエンドでの描画遅延やステータス更新が想定通りか結合テストを行います。
*   **ガードレールテスト**: 意図的に無限ループするMOCKスクリプトを実行し、タイムアウト設定によりプロセスが正しくKillされ、リソースリークが起きないことを確認します。
*   **本番シナリオ**: 問題なければ本番のClaudeCLIに簡単なテキスト編集タスクを一つ渡して、期待通りに動作・完了するか通しテストします。
