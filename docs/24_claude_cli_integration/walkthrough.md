# Epic 24: ClaudeCLIを用いたタスクの自律実行基盤の構築 (Walkthrough)

本ドキュメントでは、カンバンタスクを起点としたClaudeCLIの自律実行基盤の検証手順（テストシナリオ）と実装内容のサマリーを記載しています。

## 🎯 実装サマリー

1. **PTY基盤によるプロセス実行**:
   - バックエンド (Rust) に `claude_runner.rs` を実装し、`portable-pty` を用いて、`claude` (Anthropic Claude Code) を対話モード（エスケープシーケンスを生かした形）で安全に実行できる基盤を整備しました。
2. **リアルタイムなログ同期**:
   - Tauriのイベントシステムを利用し、PTYからの出力ストリーム (`claude_cli_output`) をフロントエンドへと即座に送信し、Reactの TerminalDock (xterm.js) 上へ遅延なく描画する仕組みを実現しました。
3. **安全機構（ガードレール）の導入**:
   - API費用枯渇や無限ループを防ぐため、**3分（180秒）の強制タイムアウト**により対象プロセスを自動停止させる機構を導入しました。
   - UI上の実行中タスクに対して、ユーザーがいつでも**「実行を強制停止 (Kill)」**できる赤いボタンをターミナル右上に追加しました。
4. **リアクティブなステータス管理**:
   - タスクの開始操作で直ちにカンバン上で `In Progress` に移動し、プロセスが正常に完了した際は自動的に `Done` に移る仕組み（およびToastによる完了/エラー通知）を整備しています。

## 🧪 実地テスト手順 (Verification Plan)

以下の手順で、新しいUIとClaudeの実地動作検証をお願いします。

### テスト1: プロセスの起動とUI遷移
1. 任意のワークスペース（Project）画面に移動します。（※ヘッダーのSettingsから `Local Path` が正しく設定されていることを事前に確認してください）
2. Kanbanボードの「To Do」列にある任意のタスクカードにマウスカーソルを合わせます。
3. カード右上の「ケバブメニュー（︙）」の横に出現する、「**開発を実行（ターミナルアイコン）**」ボタンをクリックします。
4. **検証事項**: 
   - 通知（Toast）で「Claudeでの開発を開始しました」と表示されるか。
   - タスクが即座に `In Progress` 列へ移動するか。

### テスト2: ターミナルのリアルタイム出力とUI
1. タスクの開始後、出力がバックグラウンドに流れます。
2. もしTerminal Dock（画面下部）が最小化されている場合は、「Dev Agent Terminal」のバーをクリックしてターミナル領域を展開してください。
3. **検証事項**: 
   - `claude` CLI の初期化ログや、与えたプロンプトをもとにした推論・実行ログがターミナル上にリアルタイムに（色付きで）流れるか。
   - タスクがバックグラウンドで動いている間、Terminal Dockの右上に赤い **「実行を強制停止」** ボタンが表示されているか。

### テスト3: 手動キルボタンによるガードレール検証
1. プロセスの出力が流れている最中に、Terminal Dock内に出現した「**実行を強制停止**」ボタンをクリックします。
2. **検証事項**:
   - クリック直後、UI上に「強制終了シグナルを送信しました」というToast通知が出るか。
   - スレッドが絶たれ、ターミナル上に「Process Exited: Manually killed by user」という赤い終了ログが出力されるか。
   - その後、強制停止ボタンがUI上から消えるか。

### テスト4: タイムアウトの検証 (※任意)
- 万一放置してしまった場合でも、実行開始から3分（180秒）経過すると、自動的に「Timeout reached」としてKillされる裏側の機構が働くようになっています。
