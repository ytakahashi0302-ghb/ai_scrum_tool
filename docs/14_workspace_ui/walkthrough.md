# Epic 2: フェーズ3 フロントエンド ワークスペースUI実装 - Walkthrough

## 1. 変更内容 (What changed)
フロントエンドのカンバンボードに対して、Tauriバックエンド経由で動作する「ワークスペース（プロジェクト）の切り替えUI」および「動的なデータフィルタリング」機能を実装しました。

### 主要な実装ポイント
- **WorkspaceContextの導入**: アプリケーション全体で選択中のプロジェクトID (`currentProjectId`) を管理するReact Contextを新設。
- **UI コンポーネントの追加**:
  - `ProjectSelector.tsx`: ヘッダー部分に配置されたドロップダウンメニュー。既存のプロジェクト一覧から切り替え可能。
  - `CreateProjectModal.tsx`: 「新規ワークスペース作成」を選択した際に立ち上がる入力フォーム。
- **データ取得フックの動的化**: `useStories`, `useTasks`, `useSprintHistory`, `useSprintArchive` の各カスタムフック内でハードコードされていた `projectId: 'default'` を `currentProjectId` に置換。これにより、選択中のプロジェクトに基づくデータの動的更新（リアクティビティ）を実現しました。

## 2. テスト手順 (How to verify)

POにご確認いただきたい手順は以下の通りです。現在バックグラウンドで起動中の開発サーバー (`npm run tauri dev`) にて検証を行ってください。

### Step 1: 初期表示と既存データの確認
1. アプリケーション画面を開き、ヘッダーのロゴ (`MicroScrum AI`) の右側に「default」または最初のワークスペース名が表示されていることを確認します。
2. ボード上に従来のフェーズで作成した既存のストーリーやタスクが正常に表示されているか確認します。

### Step 2: 新規ワークスペースの作成
1. ヘッダーのワークスペース名（ドロップダウンボタン）をクリックし、メニューを展開します。
2. ドロップダウンの一番下にある「新規ワークスペース作成」をクリックします。
3. モーダルが立ち上がったら、「ワークスペース名」に適当な名前（例: `マイクロスクラム開発 v2`）を入力し、「作成」ボタンをクリックします。
4. **確認事項**:
   - `react-hot-toast` による「成功」の通知が表示されるか。
   - 作成直後、ヘッダーのプロジェクト名が今作成したものに自動で切り替わるか。
   - ボードの表示が瞬時に切り替わり、**ストーリーやタスクが空の状態の新しいカンバン**が表示されるか（リロード不要でリアクティビティが効いているか）。

### Step 3: プロジェクトの切り替え動作
1. 再度ヘッダーのドロップダウンメニューを開きます。
2. 作成したプロジェクトと、元のプロジェクトの両方がリストに存在することを確認します。
3. リストから元のプロジェクトをクリックして選択します。
4. **確認事項**:
   - ボードの表示が瞬時に切り替わり、元のタスクとストーリーが正しく復元されるか。
   - `npm run lint` が修正後 0 errors で通っているか（先ほどのステップでAI側にて完了確認済み）。

### Step 4: [追加実装] タイマーの状態隔離と安全機構の連携
1. スプリントタイマーの独立化:
   - スプリントタイマーを開始し、別のプロジェクトに切り替えた際、タイマーの状態がプロジェクトごとに独立して管理されること（`sprintState_${projectId}` としてSQLite/Tauri Storeに保存）を確認します。切り替え先のプロジェクトでは新規タイマーとして初期化されます。
2. Interaction Guard:
   - タイマーが稼働中の状態で既存の別プロジェクトへ切り替えようとすると、「タイマーを一時停止して切り替えますか？」という確認ダイアログが表示されることを確認します。
3. **既知のバグ (Tech Debt)**:
   - タイマー稼働中に「新規ワークスペースを作成」し、遷移した場合、上記の Interaction Guard が発動せずタイマーが動き続けたまま新プロジェクトへ引き継がれるというエッジケースのバグが存在します。これはレアケースとして許容し、`Rule.md` に技術的負債として記録しました。
