# Epic 5: AI開発チーム (AI Team Leader) 実装計画 (Pivot版)

※ アーキテクチャのブロッカー（スクラム基盤の未分離）が発見されたため、本Epicは一時凍結中（Paused）です。

タスク単位のAIチャットから、プロジェクト全体を俯瞰する「AIチームリーダー」との対話へと方向性をピボット（再構築）します。不要となったタスク単位のレイアウトとデータをロールバックし、スプリント/プロジェクトレベルの新しいチャット機能を提供します。

## 背景と目標
- **課題**: タスクごとに個別にAIとチャットするUXは、操作が煩雑になり人間の管理コスト（マイクロマネジメント）が高すぎる。
- **解決策**: 複数のタスクやスプリント全体を視界に収める「スクラムマスター 兼 リードエンジニア」としてのAIチームリーダーを配置する。
- **目標**:
  1. `TaskFormModal` の2カラムレイアウトを元のシンプルなフォームへロールバック。
  2. スプリントボード画面（`Board.tsx`）の右側に開閉可能なサイドパネルを追加し、AIチームリーダーとのチャットインターフェースを作成する。
  3. チャット履歴をプロジェクトレベル（`team_chat_messages` テーブル）で保存。
  4. RAGコンテキストを拡張し、AIが「現在のスプリント内の全タスク構造」を把握し、自律的にレビューや割り振り提案ができるようにする。
