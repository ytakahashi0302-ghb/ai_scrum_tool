# 最終ポーリッシュ計画 (Final UI Polish Plan)

## 背景と目的
`handoff.md` の指示に基づき、AI Scrum Tool をプロトタイプ段階から「完成されたプロダクト」へ昇華させるための最終フェーズを実施します。UIのリアクティビティ確保（Rule.md）やこれまでの要件を満たしつつ、デザインの統一と不要なコードの整理、エラーハンドリングの強化を行います。

## User Review Required
> [!IMPORTANT]
> 以下の修正方針についてレビューをお願いいたします。承認 (GOサイン) をいただき次第、実際のコード改修（EXECUTIONフェーズ）を開始します。

## Proposed Changes

### 1. フルHD最適化とレイアウト調整 (追加要件)
- **対象**: `src-tauri/tauri.conf.json`, `KanbanBoard`, 各種モーダル
- **方針**: 初期起動ウィンドウサイズを 1920x1080 に変更。カンバンの `max-w` 制約を解除し、大画面でも各カラムが均等に広がるように調整する。またモーダルが小さくなりすぎないよう `max-w-2xl` 以上の適切な拡幅を行う。

### 2. 開発用ツールの整理・非表示化
- **対象**: `DeveloperTools` (Mock Data追加ボタンなど)
- **方針**: Viteの環境変数 `import.meta.env.DEV`等を利用し、開発環境（`npm run tauri dev`）でのみ表示され、プロダクションビルド時にはUIから完全に非表示になるように制御します。

### 3. UIデザインの統一と微調整
- **対象**:
  - 全てのボタン (`button`要素や汎用コンポーネント)
  - モーダル群 (`TaskFormModal`, `StoryModal`, `HistoryModal`)
  - カンバンボードのカラム (`TO DO`, `IN PROGRESS`, `DONE`)
- **方針**:
  - Tailwind CSS のクラスを見直し、ボタンカラー（例: インディゴ/ブルー系ベース）とホバー時のエフェクト（輝度変化・トランジション）を統一します。
  - モーダルの角丸設定 (`rounded-xl` など) を統一し、洗練された印象を与えます。
  - 余白 (`gap`, `p`, `m`) の一貫した適用を確認し、UIの詰まりすぎや空きすぎを解消します。
  - カンバンボードの列背景色にわずかな色差 (`bg-gray-50`/`bg-slate-50`など) を設け、列の境界をより視覚的に区別しやすくします。
  - カンバンボードの縦方向のスクロールバーを目立たなくする（`scrollbar-hide` 等）、またはモダンなカスタムスクロールバーに調整します。

### 3. エラーハンドリングとエッジケースの点検
- **対象**:
  - 各種フォームの保存アクション (`useStoryCrud`, `useTaskCrud` の呼び出し元)
  - データ保存処理全体
- **方針**:
  - Story 作成/編集時にタイトルが未入力（または空白のみ）の状態で Save した場合に、処理を中断し視覚的なバリデーションを適用します。
  - Task 作成/編集時にも同様の必須項目チェックを行います。
  - エラーの視覚的なフィードバックとして、`react-hot-toast` を用い `toast.error('タイトルを入力してください')` のような通知を網羅的に配置します。
  - その他、予期せぬDBエラー等の際もサイレントエラーにならないよう、トーストによるエラーフィードバックが行われているかコードベースを点検・補完します。

### 4. UIの日本語化 (Localization)
- **対象**: `App.tsx`, `SprintTimer.tsx`, `Board.tsx`, `StorySwimlane.tsx`, `StatusColumn.tsx`, 各種 Modal コンポーネント
- **方針**:
  - ハードコードされている英語のラベルを日本語に変更します。（例："Sprint Board" → "スプリントボード" など）
  - **重要（DBとUIの分離）**: タスクのステータス（`To Do`, `In Progress`, `Done`）はDBの保存値やドラッグ＆ドロップロジックでキーとして使用されているためそのまま維持し、「表示上のラベル」のみを日本語（未着手 / 進行中 / 完了）にマッピングして表示します。
  - トースト通知（`react-hot-toast`）のメッセージも日本語化します。

### 5. タスクカードのドラッグUX改善
- **対象**: `TaskCard.tsx`, `Board.tsx`
- **方針**:
  - `dnd-kit` の `listeners` と `attributes` を、現在のドラッグハンドルアイコンから `TaskCard` のルート要素（一番外側の `div`）に適用し、テキスト以外の空白部分など「どこを掴んでもドラッグできる」ようにします。
  - モーダルを開くイベントとドラッグが競合しないよう、`Board.tsx` の `PointerSensor` にアクティベーション制約（例: `activationConstraint: { distance: 5 }`）を追加します。
  - 不要になったドラッグハンドルアイコンは削除し、デザインと余白をシンプルに整えます。

## Verification Plan
### Automated Tests
- フロントエンドの E2E/ユニットテスト機構は現在未導入のため、手動テストを主とします。

### Manual Verification
1. **開発ツールの非表示確認**: 
   - コード上で一時的に `import.meta.env.DEV` を `false` 相当の条件にして、DeveloperToolsが消えることを確認。
2. **デザイン統一の視認**:
   - アプリを起動し、各種ボタンのホバー時の挙動、モーダルの角丸がすべての画面で統一されていることを目視確認。
   - カンバンボードの列の背景色、スクロールバーの視認性を確認。
3. **エラーハンドリングのテスト**:
   - UI上から新規Story作成時、タイトルを空にして「Save」をクリックし、保存されずにToastエラーが出ることを確認。
   - 新規Task作成時も同様に必須項目制約が効くことを確認。
   - 可能であれば意図的にDBエラーを発生（テーブル名変更など）させ、エラー時のトーストが正しく表示されることを確認。
