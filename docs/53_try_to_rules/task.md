# EPIC53: Try → ルールパイプライン

## 背景

レトロスペクティブで合意されたTryアイテムは、次のスプリント以降のエージェント動作に反映されるべきである。承認されたTryを「ルール」として永続化し、エージェントのタスク実行プロンプトに自動注入する仕組みを構築する。ルールの管理UIもSettingsPageに追加する。

## ゴール

- RetrospectiveViewでTryアイテムを「ルール化」できるUIを実装する
- エージェントのタスク実行プロンプトにアクティブルールを自動注入する
- SettingsPageにルール管理画面を追加する（有効/無効切替、編集、削除）

## スコープ

### 含む

- RetrospectiveViewのTryカラムに「ルール化」ボタン追加
- `src-tauri/src/claude_runner.rs` の `build_task_prompt` / `execute_claude_task` でルール注入
- `src/components/ui/settings/` にルール管理セクション追加
- ルールの有効/無効切替、編集、削除UI

### 含まない

- ルールの自動提案（将来的にSMが提案する拡張は別EPIC）
- ルールの優先度付け（初期実装では全アクティブルールを均等に注入）

## タスクリスト

### Story 1: ルール化UI（RetrospectiveView）

- [ ] Tryカラムの承認済みアイテムに「ルール化」ボタン追加
- [ ] ボタンクリック → `useRetroRules` の `addRule()` 呼び出し
- [ ] ルール化済みの視覚的フィードバック（バッジ or 色変更）
- [ ] 確認ダイアログ（ルール内容のプレビュー + 編集可能）

### Story 2: プロンプトへのルール注入（バックエンド）

- [ ] `execute_claude_task` 内で `get_retro_rules(project_id)` を呼び出し
- [ ] アクティブルールを `additional_context` に結合
- [ ] `build_task_prompt` で「過去のレトロスペクティブからのルール」セクションとして表示
- [ ] ルールが0件の場合は何も追加しない

### Story 3: ルール管理UI（SettingsPage）

- [ ] SettingsPage内に「レトロルール」セクション追加
- [ ] ルール一覧表示（内容、作成元スプリント、ステータス）
- [ ] 有効/無効トグルスイッチ
- [ ] ルール内容のインライン編集
- [ ] ルール削除（確認付き）
- [ ] 新規ルール手動追加フォーム

## 完了条件

- [ ] エージェントのタスク実行時にアクティブルールがプロンプトに含まれる
- [ ] SettingsPageでルールの一覧・有効無効切替・編集・削除ができる
- [ ] `cargo test` / `npm run build` がエラーなく完了する
