# Walkthrough: Epic 44 - インセプションデッキ作成ウィザードの改善（設計編）

## 概要
本取り組みは、ユーザーがプロジェクトの初期構想（インセプションデッキ）を、より自然な対話を通じてAIと共に策定できるようにするための設計変更です。

## 調査内容と設計のポイント

### 現状の分析
- `src-tauri/src/ai.rs` 内のシステムプロンプトが「箇条書きのみ」「挨拶不要」といった強い制約を設けていたため、AIがユーザーの入力に対して深掘りしたり、例を提示したりすることが困難な状態でした。
- プロダクトの目的を簡潔に定義する「エレベーターピッチ」の枠組みがなかったため、AIがプロダクトの主要な対象や価値を曖昧に理解するリスクがありました。

### 改善後の期待される挙動
1.  **親和性の高い対話**: ユーザーが「何を作ればいいか分からない」状態からでも、AIが「それは誰のためのアプリですか？」や「既存のサービスと何が違いますか？」といったガイドを出すようになります。
2.  **エレベーターピッチの自動生成**: 対話を通じて得られた情報を基に、標準的なエレベーターピッチの形式でドキュメントが構成され、プロジェクトの「北極星」として機能するようになります。
3.  **専門用語の隠蔽**: 「インセプションデッキ」「コンテキスト管理」といった用語の解説に時間を割くのではなく、プロダクトそのものの価値を語ることに集中できるUXを提供します。

## 影響範囲の確認
- `PRODUCT_CONTEXT.md` のフォーマットが変更されるため、既存のプロジェクトでインセプションデッキを再生成した際に、過去のセクション構成とマージされる挙動を確認する必要があります。
- AIのトークン消費量に配慮しつつ、適切な対話の自由度を確保しました。
