# Walkthrough: Epic 44 - インセプションデッキ作成ウィザードの改善

## 概要
Epic 44 では、インセプションデッキ作成ウィザードを「専門用語を知っている前提の対話」から、「プロダクトの価値を一緒に言語化する対話」へ寄せる改善を行いました。
あわせて、PO 追加要件として AI 連携の既定モデルを最新構成へ更新しました。

## 実装ハイライト

### 1. デフォルトモデルの最新化
- Gemini 系の既定値を `gemini-3-flash-preview` に統一しました。
- Codex / OpenAI 系の既定値を `gpt-5.4-mini` に更新しました。
- Claude 系の既定値を `claude-haiku-4-5` に更新しました。
- 対象は CLI Runner、API provider fallback、設定 UI の初期値、チームテンプレート初期値、観測系の価格テーブルです。

### 2. バックエンド改善 (`src-tauri/src/ai.rs`)
- `build_inception_system_prompt` のロールを `Scrum Product Partner` に変更しました。
- 一問一答や 1 文固定のような厳格制約を外し、短いガイドや記入例を返せるようにしました。
- Phase 1 の `PRODUCT_CONTEXT.md` テンプレートに `## エレベーターピッチ` を追加しました。
- エレベーターピッチに必要な「ターゲット」「課題」「解決策」「主要な価値」「差別化ポイント」を明示的に聞き出すようにしました。
- プロンプト改修を守るため、関連ユニットテストを追加しました。

### 3. フロントエンド改善 (`src/components/project/InceptionDeck.tsx`)
- `PHASE_GUIDE_MESSAGES` を、専門用語中心の文言から具体的な問いかけ中心の文言へ変更しました。
- フェーズ再移動時のメッセージも同じトーンに揃え、途中で戻っても迷いにくいようにしました。
- UI 構造の追加変更は行わず、既存導線のまま文言改善で体験を向上させました。

### 4. Phase 3 の平易化フォローアップ
- Phase 3 を「技術スタックを答えるフェーズ」ではなく、「どんな環境でどう使いたいかを話すフェーズ」に再定義しました。
- 非ITユーザーが技術名を答えられなくても、PC/スマホ、ローカル/クラウド、通知やデータ移行の希望だけで進められるようにしました。
- バックエンド側でも、ユーザーが技術希望を自分から出していない限り、フレームワーク名の深掘りをしない方針を追加しました。

### 5. CLI Scaffolding の実行安定化フォローアップ
- `create-next-app` などの CLI scaffolding を、Inception ドキュメントが既に存在するプロジェクト直下で直接実行しないようにしました。
- 一時ディレクトリ上で scaffold を実行し、生成された成果物だけをプロジェクト直下へ取り込むことで、`PRODUCT_CONTEXT.md` や `ARCHITECTURE.md` と共存できるようにしました。
- Next.js の scaffold コマンドには `--yes` `--skip-install` `--disable-git` を追加し、対話待ちや不要な副作用を減らしました。
- Scaffolding エラー時も実行ログを UI 上に残すようにし、再発時の切り分けをしやすくしました。

### 6. Inception Deck の POアシスタント表示
- Inception Deck のチャットフェーズと Scaffolding フェーズの両方に、POアシスタントのアバター付きバッジを追加しました。
- Kanban 画面の POアシスタント導線と同じ設定画像を使い、どの画面でも「誰が伴走しているか」が分かるようにしました。

### 7. Inception Deck チャット画面の立ち絵表示
- POアシスタントチャットと同じ見え方に寄せるため、Inception Deck の Phase 1〜4 チャット画面にも POアシスタントの立ち絵を追加しました。
- 右下に常駐する形で表示しつつ、メッセージ領域には追加の右余白を設けて、会話本文と干渉しにくいレイアウトにしています。
- 画像読み込みに失敗した場合は自動で非表示に切り替わるため、表示崩れを避けられます。

### 8. Dev ターミナルの初期ログ重複抑止
- Windows の Dev エージェント実行では、CLI の `stdout` と `stderr` を同じ terminal stream に流していました。
- Gemini CLI / Codex CLI が同一メッセージを両ストリームへ出した場合、そのまま 2 回表示されていたため、短時間に同一内容が来たときだけ重複を抑止する仕組みを追加しました。
- 影響範囲を絞るため、この抑止は Dev エージェントの CLI ストリーミング経路に限定しています。

### 9. Claude Code CLI の表示方式調査
- 調査の結果、現在の実装は Claude Code CLI を `claude -p ... --verbose` の headless 実行で呼び出しています。
- 公式ドキュメント上、`-p` は非対話モードで「最終結果を出力する」用途が中心です。
- 一方で、Claude Code には `stream-json` と partial message を使う別の出力モードがあり、途中経過を細かく拾いたい場合は現在の呼び出し方とは別設計が必要です。
- 今回は調査のみとし、実装変更は行っていません。

### 10. Epic 44 のクロージング
- PO による手動確認とレビューが完了し、インセプションデッキの対話改善、Scaffolding 安定化、POアシスタント UI 追加まで含めて Epic 44 の成果として受け入れられました。
- 次 Epic 向けに `handoff.md` を新規作成し、UX 改善が続く前提と、Claude Code CLI の `stream-json` 活用改善候補を引き継げる状態にしました。
- `BACKLOG.md` には、Claude Code CLI の部分メッセージ表示改善タスクを明示的に追加しました。
