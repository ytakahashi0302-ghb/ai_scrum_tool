# Implementation Plan: Epic 44 - インセプションデッキ作成ウィザードの改善

## 1. 背景と課題
現在のインセプションデッキ作成ウィザードは、AIの指示（システムプロンプト）が厳格すぎて、ユーザーとの対話が機械的（一問一答形式）になりがちです。また、スクラムやインセプションデッキの専門用語が前面に出ているため、これらに詳しくないユーザーにとっては何を答えればよいか戸惑うケースがあります。
さらに、プロダクトの根幹となる「エレベーターピッチ」の概念が欠落しており、プロダクトのコンテキストをAIに理解させるための情報が不足しています。

## 2. 目標
- AIのロールを「プロダクトパートナー」とし、ユーザーの意図を汲み取る柔軟な対話を実現する。
- 専門用語を避け、プロダクトの価値や課題にフォーカスしたガイドを行う。
- `PRODUCT_CONTEXT.md` に「エレベーターピッチ」セクションを追加し、プロダクトの核となるメッセージを明文化する。

## 3. 修正方針

### 3.1 バックエンド (`src-tauri/src/ai.rs`)
- **`build_inception_system_prompt` の改修**:
    - AIの役割定義を「Inception Deckファシリテーター」から「Scrum Product Partner」に変更。
    - 出力制約（箇条書きのみ、1文のみ等）を緩和し、ユーザーへの適切なアドバイスや例示を許可する。
    - Phase 1 の `PRODUCT_CONTEXT.md` 生成テンプレートに「## エレベーターピッチ」セクションを追加。
    - エレベーターピッチの構成要素（ターゲット、課題、解決策、競合優位性など）をヒアリングするよう指示を強化。

### 3.2 フロントエンド (`src/components/project/InceptionDeck.tsx`)
- **`PHASE_GUIDE_MESSAGES` の改善**:
    - Phase 1: 「プロダクトのコア価値」という表現から、「このアプリで誰のどんな課題を解決したいですか？」といった、より具体的な問いかけに変更。
    - Phase 2: 「やらないことリスト」の重要性を補足するメッセージに変更。

## 4. テスト方針
- **対話テスト**:
    - 専門用語を使わずにプロダクトのアイデアを伝え、AIが適切にエレベーターピッチを構成できるか確認する。
- **ドキュメント生成テスト**:
    - `PRODUCT_CONTEXT.md` に追加されたエレベーターピッチセクションが正しく保存されるか確認する。
- **AIコンテキスト連携テスト**:
    - 生成された `PRODUCT_CONTEXT.md` の内容に基づいて、その後のバックログ作成が精度良く行われるか（プロダクトの趣旨に沿っているか）を確認する。
